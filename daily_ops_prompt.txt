# QuantMacro Daily Operations Prompt (OpenClaw)

## Mission
Run the local daily pSEO pipeline in fail-fast mode. Do not manually edit content files during this run.

## Command
```bash
python3 scripts/macro_pipeline/run_daily_ops.py --project-root . --strict --push
```

## Pipeline Contract
1. Fetch/update data
2. Compute risk metrics (on runtime DB copy)
3. Build changed targets and generate pages
4. Generate sitemap
5. Run quality gates
6. Run astro build
7. Generate KPI report
8. Commit/push only whitelisted files when changes exist

## Fail-Fast Rules
- Stop immediately when a required step fails.
- Write error details to `logs/daily_ops/YYYY-MM-DD.json`.
- Do not push when any required step fails.

## Git Whitelist
- `src/content/blog/`
- `public/sitemap.xml`
- `src/daily_snapshot.json`
- `data/page_manifest.json`

## Notes
- The system is educational content only and should avoid trading advice language.
- Keep output deterministic and avoid generating `nan%` values.
- Runtime artifacts (`node_modules`, `.astro`, `dist`, local DB runtime noise) must not block whitelist push.
